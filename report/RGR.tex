\section{1 Введение}

Целью курсового проекта является проектирование интерфейса и программная реализация клиент-серверного SPA-приложения для стриминга и обмена видеоконтентом. Проект представляет собой полноценную экосистему, разделенную на изолированный REST API (бэкенд) и интерактивный визуальный интерфейс (фронтенд) с адаптивной версткой под десктопные и мобильные платформы.

\subsection{1.1 Задачи проекта}
\begin{itemize}
    \item Разработка архитектуры базы данных для хранения профилей, видеофайлов и интерактивных связей (лайки, подписки, комментарии).
    \item Реализация REST API эндпоинтов с безопасной JWT-аутентификацией.
    \item Проектирование современного UI/UX интерфейса в темной цветовой палитре средствами Tailwind CSS.
    \item Интеграция клиентской и серверной частей через асинхронные HTTP-запросы.
    \item Развертывание (деплой) готового веб-приложения на удаленный VDS-сервер под управлением связки Nginx и Gunicorn.
\end{itemize}

\subsection{3.1 Модуль JWT-аутентификации пользователей}
Для защиты персональных данных спроектирована форма регистрации и авторизации. При отправке данных фронтенд получает связку Access/Refresh токенов, сохраняет Access-токен в \texttt{localStorage} и перенаправляет пользователя в личный кабинет через \texttt{AuthContext}. Дизайн форм выполнен в минималистичном стиле с валидацией ошибок ввода.

\section{4 Развертывание приложения на VDS-сервер}

Проект успешно перенесен из локальной среды разработки в продуктивную (Production) на удаленный VDS-сервер с IP-адресом \texttt{62.109.1.29}. 

В ходе деплоя были настроены следующие системные компоненты:
\begin{itemize}
    \item \textbf{Gunicorn WSGI} --- обеспечивает стабильное исполнение процессов Python и работу Django-бэкенда в фоновом режиме на порту \texttt{8000}.
    \item \textbf{Nginx Web Server} --- выступает в роли обратного прокси-сервера (Reverse Proxy). Он перехватывает все внешние HTTP-запросы на порт \texttt{80}, распределяет статические файлы скомпилированного React-приложения (\texttt{frontend/build}) и перенаправляет запросы к эндпоинтам бэкенда на локальный порт Gunicorn.
\end{itemize}

\section{5 Трудности, их решение и выводы}

В ходе проектирования UI/UX интерфейса и связывания его с API возникли следующие проблемы:
\begin{itemize}
    \item \textbf{Блокировка CORS} --- при попытке отправить токены авторизации браузер отклонял запросы. Проблема решена тонкой настройкой CORS-заголовков в бэкенде.
    \item \textbf{Конфликт миграций модели User} --- расширение стандартных полей пользователя ломало БД. Решено очисткой старых схем и перегенерацией миграций.
    \item \textbf{Ошибка доступа Nginx (403 Forbidden)} --- сервер не мог прочитать бинарные медиафайлы из папки root. Решено корректировкой прав доступа веб-сервера.
\end{itemize}

В результате расчетно-графического задания спроектирована современная видеоплатформа «MyTube». Все задачи, включая JWT-авторизацию, динамический поиск, загрузку контента через модальные окна и кастомизацию профилей Tailwind-компонентами, выполнены в полном объеме. Приложение развернуто на удаленном сервере и готово к работе.
